ℹ️ Generative Adversarial Nets (GAN) Goodfellow et al., 2014 (NeurIPS)

# 📖 Abstract
ℹ️ 이 논문은 컴퓨터가 진짜 같은 데이터를 스스로 만들어낼 수는 없을까?라는 문제의식에서 출발한 연구라고 느꼈다.
기존의 생성모델은 데이터 분포를 학습하기 위해 복잡한 확률 계산이나 MCMC같은 무거운 과정이 필요한 경우가 많았고
학습 또한 쉽지 않았다. 이를 해결하기 위해 저자들은 두 개의 신경망이 서로 경쟁하면서 학습하는 구조를 제안하였다.
하나는 가짜 데이터를 만들어내는 Generator, 다른 하나는 그 데이터가 진짜인지 가짜인지 판별하는 Discriminaotr이다.
Generator는 점점 더 진짜 같은 데이터를 만들려고 하고 Discriminator는 그 가짜를 더 잘 잡아내려고 한다.
이런 경쟁 과정을 반복하면 결국 Generator는 실제 데이터 분포와 매우 비슷한 샘플을 만들어낼 수 있게 된다.
논문에서는 이 구조를 minimax game으로 수식화하였고 이론적으로 Generator가 실제 데이터 분포를 완전히 따라가면
Discriminator는 더 이상 진짜와 가짜를 구별할 수 없어 모든 입력에 대해 1/2의 확률을 출력하게 된다고 설명한다.
또한 실험을 통해 손글씨 숫자, 얼굴, 자연 이미지와 같은 데이터에서 그럴듯한 샘플을 생성할 수 있음을 보였다.
이 연구는 이후 GAN 계열 연구의 출발점이 되었다는 점에서 매우 중요한 의미를 가진다.

# 📚 Background
ℹ️ 
## 📍 Related Work 1: Restricted Boltzmann Machines(RBM), Deep Boltzmann Machines(DBM), Deep Belief Networks(DBN)
GAN이 나오기 전에도 생성모델 연구는 활발히 진행되고 있었다. 대표적으로 RBM, DBM, DBN과 같은 확률적 생성모델이 있었다.
이 모델들은 데이터의 분포를 학습하여 새로운 샘플을 생성할 수 있다는 점에서 의미가 컸다.
하지만 이런 모델들은 학습 과정에서 확률분포의 정규화 상수를 계산해야 하거나 이를 직접 계산할 수 없어 MCMC 같은 근사적 방법을 사용해야 하는 문제가 있었다.
따라서 생성모델은 가능하지만 계산적으로 복잡하고 다루기 어려운 영역으로 여겨졌다.
GAN은 이런 기존 생성모델의 복잡성을 파악하면서 판별 문제를 통해 생성모델을 학습하자는 새로운 방향을 제시했다는 점에서 차별성을 가진다.

## 📍 Related Work 2: Noise-Contrastive Estimation(NCE) / Auto-Encoding Variational Bayes(VAE 초기 흐름) / Generative Stochastic Networks(GSN)
논문에서는 NCE, GSN, variational 계열의 생성 연구들도 함께 언급한다.
NCE는 생성모델을 학습할 때 실제 데이터와 고정된 노이즈 분포를 구별하는 방식으로 학습하는 접근이다.
이 점에서 GAN과 비슷하게 판별적 학습 아이디어를 사용한다는 공통점이 있다.
하지만 GAN은 고정된 노이즈를 비교하는 것이 아니라 학습되는 Generator가 만든 샘플과 실제 데이터를 구별하도록 만든다는 점에서 훨씬 동적인 구조이다.
또한 GSN이나 variaitonal 계열 방법들은 생성이 가능하다는 장점이 있지만 추론 과정이나 Markov chain 구조가 들어가기도 하고 모델 설계가 상대적으로 복잡할 수 있었다.
이에 비해 GAN은 Generator와 Discriminator 두 신경망만으로 경쟁 학습 구조를 만든다는 점에서 직관적이고 강력한 프레임워크를 제안한 것으로 볼 수 있다.

# 🔍 Methods
### ✅ 사용된 연구 방법: Generative Adversarial Nets(GAN)
-> 새로운 생성모델 학습 프레임워크를 제안
- Generator(G)
  - 랜덤 노이즈 z를 입력으로 받아
  - 가짜 데이터 G(z)를 생성하는 모델
  - 목표는 Discriminator가 이 데이터를 진짜라고 착각하게 만드는 것
- Discriminator(D)
  - 입력 데이터 x가 들어왔을 때
  - 그것이 진짜 데이터인지, Generator가 만든 가짜 데이터인지 판별하는 모델
  - 출력은 이 데이터가 진짜일 확률로 해석

### ✅ 실험 설계:
논문은 제안한 GAN 프레임워크를 실제 이미지 데이터셋에 적용하여 생성된 샘플이 얼마나 그럴듯한지 정성적/정량적으로 평가
- 실제 데이터에서 minibatch를 뽑음
- 랜덤 노이즈에서 Generator가 가짜 샘플을 생성
- Discriminator가 진짜와 가짜를 구별하도록 학습
- 이후 Generator를 업데이트하여 Discriminator를 더 잘 속이도록 학습
- 이 과정을 반복

### 📍 모델 비교:
이 논문은 여러 기존 생성모델과 비교되는 위치에 있음
- DBN(Deep Belief Network)
- Stacked Contractive Autoencoder
- Deep GSN(Generative Stochastic Network)

- 기존 모델들
  - likelihood 근사나 MCMC가 필요할 수 있음
  - 추론 과정이 복잡할 수 있음
- GAN
  - Markov chain이 필요 없음
  - backpropagation만으로 학습 가능
  - Generator와 Discriminator의 경쟁으로 학습

# 🔍 Experiments
### ✅ 데이터셋:
- MNIST
  - 손글씨 숫자 이미지 데이터셋
- Toronto Face Database (TFD)
  - 얼굴 이미지 데이터셋
- CIFAR-10
  - 10개 클래스의 작은 자연 이미지 데이터셋

### ✅ Models:
#### Generator
- 입력: 랜덤 노이즈 𝑧
- 구조: multilayer perceptron(MLP)
- 활성화 함수: rectifier linear units와 sigmoid를 혼합 사용

#### Discriminator
- 입력: 실제 데이터 혹은 Generator가 만든 가짜 데이터
- 구조: multilayer perceptron(MLP)
- 활성화 함수: maxout 사용
- 추가적으로 dropout 적용

### ✅ Evaluation Metrics:
1. 정성적 평가(Qualitative Evaluation)
  - Generator가 만든 샘플 이미지를 직접 시각적으로 확인
  - Figures를 통해 실제 생성된 숫자, 얼굴, CIFAR 이미지 제시
  - nearest training example을 함께 보여주어 단순 암기가 아님을 보이려 함

2. 정량적 평가(Quantitative Evaluation)
  - 생성 샘플들에 Gaussian Parzen window를 맞춘 뒤
  - test set의 log-likelihood를 추정

### ✅ Implementation Details:
- 학습은 한 번에 전체 데이터를 다 쓰지 않고 일부 데이터 묶음(minibatch)을 조금씩 나누어 사용하면서 진행
- 학습할 때는 Discriminator를 먼저 학습해서 진짜와 가짜를 잘 구별하게 만들고
- 그 다음 Generator를 학습해서 Discriminator를 더 잘 속이도록 만듦
- 이 과정을 반복하면서 두 모델이 함께 성능을 높여감
- 논문 실험에서는 Discriminator를 1번 학습한 뒤 Generator를 1번 학습하는 방식(k = 1)을 사용
- Generator와 Discriminator는 모두 다층 퍼셉트론(MLP) 구조를 사용하였음
- Discriminator에는 dropout을 적용해 학습이 한쪽으로 너무 치우치지 않도록 함
- Generator는 무작위로 뽑은 랜덤 노이즈를 입력으로 받아 가짜 데이터를 생성함
- 파라미터 업데이트에는 이전 방향을 어느 정도 기억하면서 더 안정적으로 학습하는 방식(momentum)을 사용

# 🤔 Question
ℹ️
### Q1. 생성모델이란 무엇인가?
생성모델은 데이터를 분류하는 것이 아니라 새로운 데이터를 만들어내는 모델이다. 
예를 들어 손글씨 숫자 이미지나 얼굴 사진처럼 보이는 새로운 샘플을 생성할 수 있다.

### Q2. GAN의 입력 노이즈 z는 왜 필요한가?
노이즈는 다양한 샘플을 생성하기 위한 출발점이다.
서로 다른 z를 넣으면 서로 다른 생성 결과가 나오므로 Generator가 다양한 데이터를 만들 수 있다.

### Q3. 논문에서 말하는 “최적의 Discriminator”는 무슨 뜻인가?
현재 Generator가 고정되어 있을 때 진짜와 가짜를 가장 잘 구별할 수 있는 이상적인 Discriminator를 뜻한다.
논문에서는 그 형태를 수식으로 유도하였다.
