Week2 Reading Task

## 📘 Title
**Learning Phrase Representations using RNN Encoder–Decoder for Statistical Machine Translation
Cho et al., EMNLP 2014**

---

## 📖 Abstract
이 논문은 문장을 통째로 이해한 뒤 번역할 수는 없을까?라는 문제의식에서 출발한 연구라고 느꼈다. 기존의 통계 기반 기계번역은 단어와 구의 등장 빈도에 의존했기 때문에 자주 등장하지 않는 표현이나 긴 문장에서는 자연스러운 번역을 생성하는 데 한계가 있었다.
이를 극복하기 위해 저자들은 문장을 하나의 연속 벡터로 압축한 뒤 그 벡터를 기반으로 새로운 문장을 생성하는 RNN Encdoer-Decoder 구조를 제안하였다. Encoder는 입력 문장을 순서대로 읽어 하나의 요약 벡터로 만들고 Decoder는 그 벡터를 바탕으로 출력 문장을 한 단어씩 생성한다. 이 구조는 입력과 출력의 길이가 달라도 처리할 수 있다는 점에서 기존 모델과 차별화된다.
또한 기본 RNN이 긴 문장에서 정보를 잊어버리는 문제를 해결하기 위해 reset gate와 update gate를 포함한 gate 구조를 도입하였다. 이는 이후 GRU로 발전하게 되는 중요한 아이디어이다.
실험에서는 해당 모델을 기존 SMT 시스템의 추가 feature로 활용했을 때 BLEU 점수가 향상됨을 보였으며 단순한 통계 기반이 아니라 의미와 문법 구조를 반영하는 표현을 학습할 수 있음을 확인하였다. 이 연구는 이후 신경망 기반 기계번역의 출발점이 되었다는 점에서 중요한 의미를 가진다.

## 📚 Background
### 📍 Related Work 1: Neural Language Model(Bengio et al., 2003)
2003년 Bengio의 연구에서는 신경망을 활용한 언어모델이 제안되었다.
이 모델은 이전 단어들을 입력으로 받아 다음 단어의 확률을 예측하는 구조였으며 단어를 연속적인 벡터로 표현했다는 점에서 중요한 의미를 가진다.
이 접근은 단어 간 의미적 유사성을 벡터 공간에서 표현할 수 있다는 가능성을 보여주었고 이후 다양한 NLP 연구의 기반이 되었다.
그러나 이 모델은 주로 다음 단어 예측에 초점을 두었으며 서로 다른 언어 간 문장 전체를 직접 변환하는 구조는 아니었다. 따라서 번역 문제를 직접적으로 해결하기에는 한계가 있었다. 

### 📍 Related Work 2: Neural Network in SMT(Schwenk, 2012)
cho et al. 이전에도 신경망을 통계 기반 번역 시스템에 적용하려는 시도가 있었다.
예를 들어 schwenk(2012)은 feedforward neural network를 이용해 phrase 쌍에 점수를 부여하는 방식을 제안하였다.
이 방식은 번역 모델의 성능을 일부 향상시켰지만 입력 길이가 고정되어야 한다는 제약이 있었고 문장의 순서를 완전히 반영하기에는 구조적인 한계가 있었다.
이에 비해 cho et al.의 RNN Encoder-Decoder는 순차 데이터를 자연스럽게 처리할 수 있는 RNN 구조를 사용하여 입력과 출력의 길이가 서로 달라도 처리할 수 있다는 점에서 차별성을 가진다.

## 🔍 Methods
### ✅ 사용된 연구 방법:
이 논문은 RNN Encoder-Decoder 구조를 제안하고 이를 번역 문제에 적용하였다.

- Encoder는 입력 문장을 순서대로 읽어 고정 길이 벡터 c로 요약한다.
- 이 벡터는 입력 문장의 의미를 압축한 표현으로 볼 수 있다.
- Decoder는 요약 벡터 c를 기반으로 출력 문장을 한 단어씩 생성한다.

모델은 전체 문장의 조건부 확률 p(y|x)를 최대화하도록 학습된다.
또한 저자들은 기본 RNN 대신 reset gate와 update gate를 포함한 gated hidden unit을 사용하였다.
이 구조는 이전 정보를 얼마나 유지할지, 혹은 새 정보로 얼마나 업데이트할지를 조절하여 긴 문장에서 발생하는 정보 손실 문제를 완화한다.

### ✅ 실험 설계: 
제안한 RNN Encoder-Decoder는 기존 phrase-based SMT 시스템에 통합되는 방식으로 사용되었다.

- RNN이 계산한 phrase 쌍의 조건부 확률을
  -> 기존 SMT 시스템의 추가 feature로 활용
- 기존 번역 모델을 완전히 대체하지 않음
- 통계 기반 점수 + 신경망 기반 점수 결합

실험은 영어 -> 프랑스어 번역 태스크에서 수행되었으며 성능은 BLEU 점수를 통해 비교하였다.

### 📍 모델 비교:
1. Baseline
  - 기존 phrase-based SMT 시스템

2. Baseline + RNN Encoder-Decoder
  - phrase 점수에 RNN 기반 확률 추가

3. Baseline + CSLM + RNN
  - Continuous Space Language Model과 phrase scoring 결합

4. Baseline + CSLM + RNN + Word Penalty
  - 추가 regularization 요소 포함

> 결과적으로 RNN Encoder-Decoder를 추가했을 때 BLEU 점수가 향상되었으며
> CSLM과 함께 사용할 경우 가장 높은 성능을 보였다.
> 이는 제안된 모델이 기존 SMT 시스템과 상호보완적인 역할을 한다는 것을 보여준다.

## 🔍 Experiments
### ✅ 데이터셋:
실험은 WMT'14 English-French 번역 태스크를 기반으로 수행되었다.

- 학습 데이터: Europarl, News commentary, UN 코퍼스 등 대규모 병렬 말뭉치
- 데이터 선택 기법을 통해 일부 데이터 선별
- 약 3억 단어 이상의 병렬 데이터 사용
- 계산 효율을 위해 영어/프랑스어 각각 상위 15000개 단어만 vocabulary로 사용

### ✅ Models: 
#### Baseline
- Moses 기반 phrase-based SMT 시스템

#### RNN Encoder-Decoder
- phrase 쌍의 조건부 확률을 계산하여 SMT 시스템의 feature로 추가
- Encoder / Decoder 각각 1000 hidden units
- 단어 임베딩 차원: 100
- reset gate & update gate 포함(gated unit 사용)

#### CSLM(Continuous Space Language Model)
- 7-gram 기반 neural language model
- 번역 중 부분 문장을 점수화하는 데 사용

### ✅ Evaluation Metrics: 
모델의 성능 평가는 BLEU(Bilingual Evaluation Understudy) score를 사용하였다.
BLEU는 생성된 번역 문장과 정답 번역 문장 간의 n-gram 일치도를 기반으로 계산되는 지표로 기계번역 분야에서 널리 사용된다.

> Baseline 대비 RNN Encoder-Decoder를 추가했을 때 BLEU 점수가 상승하였다.
> CSLM과 함께 사용할 경우 가장 높은 성능을 보였다.

### ✅ Implementation Details: 
- Hidden unit 수: 1000
- 단어 임베딩 차원: 100
- Optimizer: Adadelta
- Mini-batch 크기: 64 phrase pairs
- 학습 시간: 약 3일
- Recurrent weight: 특수 초기화 방식 사용
- 출력층: maxout 구조
- GPU 환경에서 학습
- 확률적 경사하강법 기반 파라미터 업데이트

## 📖 Conclusion
### ✅ Limitation:
- 고정 길이 벡터의 한계: 
  Encoder는 입력 문장의 모든 정보를 하나의 고정 길이 벡터 c에 압축해야 한다.
  문장이 길어질수록 중요한 정보가 손실될 가능성이 존재한다.

- 완전한 end-to-end 번역 모델이 아님: 
  본 연구는 기존 SMT 시스템을 완전히 대체하지 않고 신경망 기반 점수를 보조 feature로 추가하는 방식에 머물렀다.

> 이러한 한계는 이후 등장한 Attention Mechnism을 통해 점차 보완되었다.

### ✅ Contribution:
1. Seq2Seq(Encoder–Decoder) 구조를 명확히 제안
2. GRU의 초기 형태가 되는 gated hidden unit 제안
3. 통계 기반 번역 시스템에 신경망 점수를 결합하여 성능 향상 입증
4. 문장 및 phrase를 의미적, 문법적으로 구조화된 벡터 공간에 표현 가능함을 실험적으로 확인

## 🤔 Question
### Q1. 논문에서 말하는 벡터는 무엇인가?
벡터는 숫자들의 리스트이다. 예를 들어 [0.2, -1.3, 0.8, ...] 같은 숫자 묶음이다. 딥러닝에서는 단어나 문장을 이런 숫자 묶음으로 바꿔서 컴퓨터가 계산할 수 있게 만든다.

### Q2. BLEU란 무엇인가?
BLEU는 번역 성능을 평가하는 점수이다. 모델이 만든 번역 문장이 정답 문장과 얼마나 비슷하지 n-gram 단위로 비교해서 점수를 계산한다. 점수가 높을수록 번역이 잘 된 것이다.

### Q3. 단어를 연속적인 벡터로 표현한다는 것은 무슨 뜻인가?
예전에는 단어를 단순한 번호처럼 다뤘다. 하지만 연속적인 벡터 표현은 단어를 여러 개의 숫자로 이루어진 벡터로 바꾸는 것이다. 이렇게 하면 단어 간의 의미 차이를 숫자로 표현할 수 있다.

### Q4. 의미적 유사성을 벡터 공간에서 표현한다는 것은 무슨 뜻인가?
비슷한 의미의 다어는 벡터 값도 비슷해진다. 예를 들어 cat과 dog의 벡터는 서로 가깝게 위치한다. 즉, 벡터 공간에서 거리가 가까우면 의미도 비슷하다고 볼 수 있다.

### Q5. Feedforward Neural Network란 무엇인가?
입력 -> 은닉층 -> 출력으로 한 방향으로만 흐르는 가장 기본적인 신경망이다. 순서 정보를 기억하지 못한다.

### Q6. phrase 쌍에 점수를 부여한다는 것은 무엇인가?
예를 들어 thank you <-> merci 이 두 문장이 얼마나 잘 대응되는지를 숫자로 평가하는 것이다. 점수가 높을수록 좋은 번역 후보로 본다.

### Q7. 입력 길이가 고정되어야 한다는 것은 무슨 뜻인가?
모델이 항상 같은 길이의 입력만 받을 수 있다는 뜻이다. 예를 들어 항상 단어 5개짜리 문장만 입력 가능하다면 3개나 10개짜리 문장은 바로 처리하기 어렵다.

### Q8. 입력 문장을 고정 길이 벡터 c로 요약한다는 것은 무엇인가?
문장을 끝까지 읽은 뒤 그 문장 전체를 하나의 숫자 벡터로 압축한다는 뜻이다. 즉, 긴 문장을 하나의 요약된 숫자 표현으로 바꾸는 것이다.

### Q9. p(y|x)를 최대화한다는 것은 무엇인가?
입력 문장 x가 주어졌을 때 정답 번역 y가 나올 확률을 최대한 크게 만들도록 학습한다는 뜻이다. 즉, 정답이 나올 가능성을 높이도록 모델을 훈련한다.

### Q10. phrase-based SMT란 무엇인가?
옛날 통계 기반 번역 방식이다. 문장을 단어 묶음(phrase) 단위로 나누고 자주 같이 등장한 번역 쌍을 통계적으로 선택하는 방식이다.

### Q11. CSLM이란 무엇인가?
Continuous Space Language Model의 약자이다. 신경망 기반 언어 모델로 문장이 자연스러운지 점수를 매기는 역할을 한다.

### Q12. WMT'14 English-French 번역 태스크란 무엇인가?
WMT는 기계번역 대회이다. 2014년에 영어-프랑스어 번역 데이터를 제공했고 많은 연구가 이 데이터로 실험을 수행했다.

### Q13. Moses 기반 phrase-based SMT 시스템이란 무엇인가?
Moses는 통계 기반 번역을 구현한 오픈소스 프로그램이다. 많은 연구에서 기본 번역 시스템으로 사용되었다.

### Q14. Encoder/Decoder 각각 1000 hidden units란 무엇인가?
RNN 내부에 1000개의 계산 노드(뉴런)가 있다는 뜻이다. 숫자가 클수록 모델이 더 복잡한 정보를 표현할 수 있다.

### Q15. 단어 임베딩 차원 100이란 무엇인가?
각 단어를 100개의 숫자로 표현한다는 뜻이다. 즉, 단어 하나 = 100차원 벡터

### Q16. CSLM의 7-gram 기반 점수화란 무엇인가?
앞의 7개 단어를 보고 다음 단어가 자연스러운지 평가하는 모델이다. 번역 결과가 자연스러운 문장인지 점수로 판단한다.

### Q17. Implementation Details 용어 설명
- Hidden unit 1000 -> RNN 내부 뉴런 개수
- Embedding 100 -> 단어 벡터 길이
- Optimizer Adadelta -> 학습 속도를 조절하는 방법
- Mini-batch 64 -> 한 번에 64개 데이터로 학습
- Recurrent weight 초기화 -> RNN 가중치 시작값 설정 방식
- Maxout 구조 -> 출력층에서 더 복잡한 표현 가능하게 하는 구조

### Q18. GPU 환경이란 무엇인가?
GPU는 그래픽카드이다. 딥러닝 계산을 빠르게 수행할 수 있어 학습 속도를 크게 높인다.

### Q19. 확률적 경사하강법(SGD)이란 무엇인가?
모델의 오류를 줄이기 위해 조금씩 가중치를 수정해 나가는 학습 방법이다.

### Q20. 파라미터란 무엇인가?
모델이 학습하면서 조정하는 숫자들이다.(예: 가중치, 편향 등)

### Q21. seq2seq 구조란 무엇인가?
sequence to sequence의 약자이다. 입력 문장(시퀀스)을 받아 출력 문장(또 다른 시퀀스)을 생성하는 구조이다.

### Q22. 문장을 벡터 공간에표현 가능함을 확인은 무슨 의미인가?
비슷한 의미의 문장들은 벡터 공간에서도 서로 가깝게 위치했다는 뜻이다. 즉, 모델이 문장의 의미 구조를 어느 정도 학습했다는 의미이다.

### Q23. 왜 고정 길이 벡터로 문장을 압축하는 방식은 한계가 있을까?
고정 길이 벡터는 아무리 문장이 길어도 항상 같은 크기의 숫자 묶음 하나로 표현해야 한다. 짧은 문장은 괜찮을 수 있지만 긴 문장은 많은 정보가 들어 있기 때문에 모든 내용을 하나의 벡터에 담는 것이 어렵다. 특히 문장의 앞부분 정보가 뒤로 갈수로 희미해질 수 있고 중요한 세부 정보가 손실될 가능성이 있다. 이러한 한계를 보완하기 위해 이후 연구에서 Attention Mechanism이 등장하였다. Attention은 문장 전체를 한 번에 압축하는 대신 필요한 부분을 그때그때 참고할 수 있게 만든 구조이다. 즉, 고정 길이 벡터 방식은 중요한 출발점이었지만 표현력에는 한계가 있는 구조라고 볼 수 있다.

### Q24. 왜 저자들은 기존 SMT를 완전히 대체하지 않고 보조 점수로만 사용했을까?
이 논문은 신경망 번역 모델을 제안했지만 기존 통계 기반 번역 시스템(SMT)을 바로 대체하지는 않았다. 그 이유는 당시에는
- 데이터 규모가 충분하지 않았고
- 계산 자원이 제한적이었으며
- 신경망 번역이 아직 안정적으로 동작한다고 보장하기 어려웠기 때문일 가능성이 크다.

또한 SMT는 이미 잘 구축된 시스템이었기 때문에 완전히 교체하기보다는 신경망 점수를 추가하여 점진적으로 개선하는 전략을 선택한 것으로 보인다. 따라서 이 논문은 완전한 전환점이라기보다는 통계 기반 번역에서 신경망 기반 번역으로 넘어가는 과도기적 단계에 위치한 연구라고 이해할 수 있다.

### Q25. Attention Mechanism은 무엇인가?
문장을 하나의 고정 길이 벡터로 모두 압축하는 대신 필요한 부분을 그때그때 참고할 수 있게 해주는 구조이다. 기존 Encoder-Decoder 구조에서는 입력 문장을 하나의 벡터 c로 요약한 뒤 Decoder가 그 벡터만 보고 번역을 생성했다. 하지만 문장이 길어질수록 중요한 정보가 벡터 안에서 희미해지는 문제가 있었다. Attention은 이 문제를 해결하기 위해 Decoder가 출력 단어를 생성할 때마다 입력 문장의 모든 단어를 다시 참고하고 그중에서 현재 가장 중요한 단어에 더 큰 가중치를 주는 방식을 사용한다.
- 기존 방식 -> 문장을 한 번에 압축해서 기억
- Attention -> 번역할 때마다 필요한 부분을 다시 찾아봄

예를 들어 영어 문장의 특정 단어를 번역할 때 그 단어와 관련된 입력 단어에 더 집중하도록 만들어주는 것이다. 즉, Attention은 모든 정보를 한 번에 담지 말고 필요할 때 필요한 정보에 집중하자는 아이디어이다. 이 구조 이후 Neural Machine Translation의 핵심이 되었고 Transformer 모델에서도 가장 중요한 개념으로 발전하였다.
